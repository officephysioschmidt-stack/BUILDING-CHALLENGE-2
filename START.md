# START.md — Anleitung für Claude Code

> **An Claude:** Diese Datei ist an dich adressiert. Der Mensch, mit dem du
> arbeitest, nimmt an der SKAILE Building Challenge teil und hat dieses Repo
> gerade aus dem Template erstellt. Führe ihn jetzt durch den Start.

## Deine Aufgaben, in dieser Reihenfolge

1. **Projekt verstehen:** Frag deinen Menschen:
   - Welches Problem willst du lösen? (Wessen Problem ist es, was kostet es
     heute an Zeit/Geld/Nerven?)
   - Wie heißt dein Projekt? (kurzer, klarer Name)
   - Baust du mit Claude Code, n8n oder beidem?
   - Was existiert dazu bei dir schon? (Vorarbeit ist erlaubt — merk sie
     dir für README und Abgabe-Post, dort wird sie ehrlich genannt.)

2. **README ausfüllen:** Trage die Antworten in die Platzhalter der README.md
   ein (Abschnitte "Das Problem", "Was der Agent macht", "Stack"). Die übrigen
   Abschnitte bleiben als Platzhalter — die werden beim Bauen gefüllt.

3. **Ersten Commit machen:**
   ```
   git add -A && git commit -m "start: [Projektname] - Challenge Tag 1"
   git push
   ```

4. **Die Regeln erklären** (kurz, in eigenen Worten):
   - EIN Agent, EIN echtes Problem. Klein anfangen.
   - Beim Arbeiten kleine Commits pushen — Richtwert: mindestens einer pro
     Bautag. Das ist das Build-Journal, nicht Überwachung.
   - Ein kurzer Update-Post pro Woche in der Community.
   - KI-Nutzung (du!) ist ausdrücklich erwünscht. Nicht okay ist nur:
     fremde fertige Arbeit als eigene ausgeben.
   - API-Keys gehören in die .env (siehe .env.example) — NIEMALS ins Repo.
   - Vorarbeit ist erlaubt — sie wird ehrlich im Abgabe-Post und in der
     README genannt ("das existierte bei mir schon"). Bewertet wird, was
     WÄHREND der Challenge entsteht.
   - Abgabe am Ende: Skool-Post mit der Abgabe-Vorlage aus ABGABE.md
     (Demo-Video 2-3 Min, EIN Durchlauf, ungeschnitten — Bildschirmaufnahme
     reicht, keine Kamera oder Stimme nötig).

5. **Loslegen:** Frag deinen Menschen, was der kleinste funktionierende
   erste Schritt seines Agenten wäre — und fang mit ihm an zu bauen.

## Hinweis für dich, Claude

Erinnere deinen Menschen im Laufe der Zusammenarbeit gelegentlich daran,
zu committen und zu pushen, wenn ihr an diesem Projekt arbeitet. Kleine,
regelmäßige Commits sind Teil der Challenge.
